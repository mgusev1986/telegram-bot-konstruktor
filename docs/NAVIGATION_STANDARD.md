# Стандарт навигации и клавиатур Telegram-бота

Единый стандарт навигации и UI для всего бота: одни и те же подписи кнопок, одни и те же цели переходов, предсказуемое поведение «Назад» / «В главное меню» / «Отмена».

---

## 0. Аудит согласованности UI (итог)

**Исправлено / приведено к стандарту:**
- Все экраны используют общие подписи: «Назад», «В главное меню», «Отмена», «Вернуться к странице», «Вернуться в админку» (i18n: back, to_main_menu, cancel_btn, return_to_page, return_to_admin).
- Корень: нет кнопки «Назад»; системный ряд: Мой кабинет → Связь с наставником → Сменить язык; у админа: Админ-панель → Настроить страницу.
- Редактор страницы и управление кнопками: одна строка Назад + В главное меню (без эмодзи у «Назад»).
- Визарды/сцены: Назад + Отмена (+ при необходимости В главное меню, Вернуться к странице, Вернуться в админку).
- Диалоги подтверждения удаления: подтвердить + Назад + Отмена; обработчики cancel_del/cancel_del_item с сообщением об отмене.
- Устаревшие действия: одно сообщение (action_stale) + клавиатура buildStaleActionKeyboard (В главное меню; у админа — Вернуться в админку).
- Единая точка входа для «В главное меню»: везде используется константа NAV_ROOT_DATA (в т.ч. в выбор раздела при переименовании/привязке).

**Не дублируются:** системные кнопки в одном порядке, без синонимов («Главная»/«Домой» не используются, только «В главное меню»).

## 1. Глобальный стандарт навигации

### Универсальные кнопки

| Действие | Подпись (i18n) | Callback / поведение |
|----------|----------------|------------------------|
| Шаг назад | `back` — «Назад» | `NAV_BACK_DATA` или кастомный callback (например `page_edit:back:{pageId}`) |
| В корень | `to_main_menu` — «В главное меню» | `NAV_ROOT_DATA` (`nav:root`) → всегда главное меню |
| Отмена потока | `cancel_btn` — «Отмена» | `SCENE_CANCEL_DATA` (`scene:cancel`) → выход из визарда/сцены |
| Вернуться к редактору страницы | `return_to_page` — «Вернуться к странице» | `page_edit:open:{pageId}` |
| Вернуться в админку | `return_to_admin` — «Вернуться в админку» | `admin:open` |

Правила:

- Одна подпись — одно действие: не смешивать «Главная» / «Домой» с «В главное меню».
- «Назад» везде одна и та же подпись `back`, без лишних эмодзи в стандартной навигации.
- В кабинете и при «устаревшем действии» для перехода в админку используется только «Вернуться в админку» (`return_to_admin`).

## 2. Типы экранов и обязательные кнопки

| Тип | Кнопки навигации | Примечание |
|-----|-------------------|------------|
| **A. Корень / главная** | Только контент + системный ряд (Кабинет, Наставник, Сменить язык) + при необходимости Админ-панель, Настроить страницу | Кнопки «Назад» на корне нет |
| **B. Вложенный контент** | Контент (если есть) + **Назад** + **В главное меню** | |
| **C. Кабинет** | Действия кабинета + **Назад** + **В главное меню**; у админа дополнительно **Вернуться в админку** | |
| **D. Админ-панель** | Действия админки + **В главное меню** | Без «Назад» |
| **E. Редактор страницы** | Действия страницы + **Назад** + **В главное меню** | |
| **F. Управление кнопками** | Действия по кнопкам + **Назад** + **В главное меню** | |
| **G. Визард / сцена** | **Назад** + **Отмена** + **В главное меню**; при необходимости **Вернуться к странице**, **Вернуться в админку** | |
| **H. Диалог подтверждения** | **Подтвердить** (или «Да, удалить») + **Назад** + **Отмена** | |

## 3. Системные кнопки (порядок и роли)

Для всех пользователей (один порядок, без дублирования):

1. **Мой кабинет** — `my_cabinet`
2. **Связь с наставником** — `mentor_contact`
3. **Сменить язык** — `change_language` (везде одна подпись, не «Выбрать язык» в одном месте и «Сменить язык» в другом)

Только для владельца/админа:

4. **Админ-панель** — `admin_panel`
5. **Настроить страницу** — `configure_page`

Они не дублируются и не показываются не той роли.

## 4. Единая система клавиатур (keyboards.ts)

- **`buildNavigationRow(i18n, languageCode, { back?, toMain })`** — одна строка: при необходимости «Назад» + при необходимости «В главное меню». Основа для всех вложенных экранов.
- **`buildStaleActionKeyboard(i18n, languageCode, isAdmin)`** — после сообщения «действие устарело»: всегда «В главное меню», для админа ещё «Вернуться в админку».
- **`buildWizardStepKeyboard(...)`** — визард: Назад, Отмена, при необходимости Пропустить; затем В главное меню; при `fromPageId` — Вернуться к странице, Вернуться в админку.
- **`buildSceneCancelBackKeyboard`** — сцены: Назад (если передан callback) + Отмена.
- **`buildReturnToAdminKeyboard`** / **`buildReturnToPageEditorKeyboard`** / **`buildReturnToAdminOrPageKeyboard`** — один или два ряда для возврата в админку/редактор страницы.
- **`buildPageEditorKeyboard`** / **`buildButtonManagementKeyboard`** — редактор страницы и управление кнопками всегда заканчиваются строкой **Назад** + **В главное меню** (без префикса «⬅️» у «Назад»).
- **`buildPageDeleteConfirmKeyboard`** / **`buildPageDeleteItemConfirmKeyboard`** — подтверждение удаления: подтвердить + **Назад** + **Отмена** (без дублирования «Нет» и «Назад»).
- **`buildParentRootKeyboard`** — выбор корня: одна кнопка с подписью из i18n `parent_root_main_menu` («Корень (главное меню)») + **Отмена**.

Константы:

- **`NAV_BACK_DATA`**, **`NAV_ROOT_DATA`** — для единообразной обработки в callback.
- **`SCENE_CANCEL_DATA`** — отмена сцены/визарда.

---

## 5. Поведение «Назад» / отмена / корень

- **«Назад»** — переход на предыдущий логический экран; если его нет — на безопасный родительский или корень.
- **«В главное меню»** — всегда корень (обработка `nav:root`).
- **«Отмена»** — полный выход из текущего потока (сцена/визард), очистка временного состояния, переход на разумный стабильный экран; при отмене удаления показывается «Действие отменено» и возврат в редактор страницы/кнопок.
- По **/start** — сброс контекста и переход в корень.
- Устаревшие кнопки: при нажатии показывается сообщение `action_stale` и клавиатура **buildStaleActionKeyboard** (В главное меню; у админа — Вернуться в админку).

## 6. Заголовки экранов (i18n)

Для единообразия можно использовать ключи:

- `screen_header_cabinet` — «💼 Мой кабинет»
- `screen_header_admin` — «⚙️ Админ-панель»
- `screen_header_page_editor` — «✏️ Редактирование страницы»

- `screen_header_language` — «🌍 Выбор языка»

При необходимости добавляется контекстная строка (например: «Текущая страница: Главная»).

## 7. Внесённые изменения (кратко)

- **Устаревшее действие:** при нажатии устаревшего callback показывается сообщение `action_stale` и клавиатура **buildStaleActionKeyboard** (В главное меню; для админа — Вернуться в админку).
- **Единая подпись «Назад»:** в редакторе страницы убрана приставка «⬅️», везде используется i18n `back`.
- **Выбор корня:** подпись кнопки выбора корня вынесена в i18n `parent_root_main_menu`.
- **Кабинет и пустое меню:** переход в админку из кабинета и с экрана пустого меню — только «Вернуться в админку» (`return_to_admin`).
- **Язык:** везде одна системная кнопка «Сменить язык» (`change_language`).
- **Диалоги подтверждения удаления:** кнопки приведены к виду Подтвердить / **Назад** / **Отмена**; добавлены обработчики `cancel_del` и `cancel_del_item` с сообщением `action_cancelled` и возвратом в редактор/управление кнопками.
- Экспортированы **NAV_BACK_DATA** и **NAV_ROOT_DATA** для единообразной обработки в register-bot.

## 8. Как тестировать навигацию в Telegram

1. **Корень:** /start → главное меню без «Назад»; системные кнопки в одном порядке; у админа — «Админ-панель» и «Настроить страницу».
2. **Вложенный контент:** зайти в раздел → есть «Назад» и «В главное меню»; «Назад» возвращает на уровень выше, «В главное меню» — в корень.
3. **Кабинет:** открыть кабинет → «Назад», «В главное меню»; у админа — «Вернуться в админку» с подписью «Вернуться в админку».
4. **Админ-панель:** только «В главное меню», без лишних кнопок назад.
5. **Редактор страницы:** «Назад» и «В главное меню» в одной строке; «Назад» без эмодзи.
6. **Визарды (новый раздел/кнопка):** на каждом шаге — «Назад», «Отмена», «В главное меню»; при входе из редактора страницы — «Вернуться к странице», «Вернуться в админку».
7. **Отмена:** в визарде нажать «Отмена» → сообщение об отмене, выход из потока. В диалоге удаления страницы/пункта нажать «Отмена» → «Действие отменено» и возврат в редактор/кнопки.
8. **Устаревшее действие:** начать визард, выйти из сцены (например в главное меню), нажать старую кнопку визарда → сообщение «Это действие устарело...» и клавиатура с «В главное меню» и (у админа) «Вернуться в админку».
9. **Язык:** в меню и в кабинете кнопка смены языка с одной подписью («Сменить язык» в RU).
10. **Выбор родителя:** в создании раздела/кнопки кнопка корня с подписью «Корень (главное меню)» и кнопка «Отмена».

После прохода по этим пунктам навигация и клавиатуры соответствуют описанному стандарту.
