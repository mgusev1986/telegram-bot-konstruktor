# Полный отчёт: аудит и доработка конструктора Telegram-бота

## 1. Full UX audit

### Где пользователь путается
- **Онбординг:** в приветствии было «главную страницу, первый раздел, первую кнопку» — создаётся впечатление, что «кнопка» отдельный шаг; по факту после раздела показывается выбор «ещё раздел или предпросмотр», а не создание «дополнительной кнопки». Исправлено: приветствие описывает шаги как язык → главная → разделы → предпросмотр и публикация.
- **Терминология:** «раздел» vs «страница» vs «кнопка» — в части экранов (мастер создания пункта меню) показываются технические поля (wizard_parent, wizard_lang, wizard_type). Для новичка полезнее явно: «страница с контентом», «кнопка, которая ведёт на страницу», «подраздел = дочерняя страница».
- **Шаг 3 онбординга:** в переводах оставался заголовок «Дополнительная кнопка» (onboarding_step3_title), хотя по коду при step === 3 показывается только выбор «Добавить ещё раздел / Предпросмотр». Текст выбора уточнён: «Добавить ещё один раздел на главной или перейти к предпросмотру».
- **Отмена:** при нажатии «Отмена» в сцене показывается «Действие отменено» и клавиатура «В главное меню» / «Вернуться в админку» — поведение корректно, сохранённые данные не сбрасываются.
- **Где нет контекста:** в редакторе страницы не везде явно видно «Вы редактируете: [название страницы]»; в сценах создания раздела/кнопки контекст родителя есть (section_hint_inside_page, page_root_title).

### Порядок шагов
- Онбординг: 0 — язык, 1 — главная, 2 — первый раздел (сцена create-section), 3 — выбор (ещё раздел / предпросмотр), 4 — предпросмотр, 5 — публикация, 6 — готово. Порядок логичный; «дополнительная кнопка» в быстром онбординге не запрашивается.
- Создание раздела: сначала название кнопки, затем контент одним сообщением — соответствует требуемой логике.

### Перегрузка экранов
- Админ-панель: много кнопок в одном списке (структура, предпросмотр, добавить пункт, рассылки, экспорт, публикация, обнулить). «Полностью обнулить бота» вынесен в явно опасное действие (подпись с ⚠️).

### Итог UX (действия)
- Привести все подписи навигации к одному стандарту с эмодзи (↩️ Назад, 🗂 В главное меню, 🛠 Настроить страницу, 💼 Мой кабинет, ☎️ Связь с наставником, 🌍 Сменить язык, 🛠 Админ-панель) — сделано в i18n.
- Онбординг: приветствие и шаг 3 описаны без «дополнительной кнопки», акцент на «разделы → предпросмотр → публикация».
- Кнопка полного обнуления помечена как опасная (⚠️).

## 2. Full routing / navigation audit

### Граф навигации (по коду)
- **Узлы:** root (виртуальная главная), все MenuItem (страницы/разделы/кнопки SECTION_LINK).
- **Рёбра:** parent → child (getMenuItemsForParent); SECTION_LINK → targetMenuItemId; «Назад» → parentId (или root); «В главное меню» → root.
- **Точки входа:** `/start` → sendRootWithWelcome; callback `nav:root` → sendRootWithWelcome; `menu:open:<id>` → sendMenuPage(ctx, id); `menu:back` с value → sendMenuPage(ctx, value) (родитель).

### Проверенные сценарии
- `/start` открывает реальную главную: getWelcome + getMenuItemsForParent(null) + buildMenuKeyboard — да.
- «В главное меню» (NAV_ROOT_DATA) ведёт в sendRootWithWelcome — тот же root.
- «Назад» при menu:back:<parentId> ведёт в sendMenuPage(ctx, parentId) — родитель с контентом.
- Редактор страницы: после удаления страницы (confirm_del) раньше вызывался sendRootMenu(ctx, parentId), что показывало только заголовок «Подменю» и список детей без контента родителя. Исправлено: после удаления вызывается sendMenuPage(ctx, parentId) — показ полной страницы родителя.
- Открытие раздела/подраздела: sendMenuPage загружает getMenuItemContent и getMenuItemsForParent — контент и кнопки отображаются.
- SECTION_LINK: при menu:open:<linkId> в register-bot проверяется type === SECTION_LINK и targetMenuItemId, показывается контент целевой страницы с parentId ссылки — корректно.

### Найденные и исправленные ошибки
- После удаления страницы (confirm_del): переход на sendMenuPage(ctx, item.parentId ?? null) вместо sendRootMenu — пользователь попадает на полную страницу родителя, а не на пустой список.

### Рекомендации
- Все целевые pageId для кнопок должны существовать в БД; иначе getMenuItemContent выбросит. Аудит навигации (runNavigationAudit) и тесты выявляют BROKEN_BUTTON_TARGET и ORPHAN_PAGE.

## 3. Persistence / state safety audit

### Что сохраняется сразу (в БД)
- Язык: setWelcome, ensureActiveTemplate, setOnboardingStep — при выборе языка в онбординге и при шаге 1.
- Главная: setWelcome(actorUserId, languageCode, text, mediaType, mediaFileId) — при принятии контента шага 1 онбординга и при редактировании главной.
- Разделы и кнопки: createMenuItem — одна транзакция (страница + локализация); порядок, parentId, targetMenuItemId сохраняются.
- Онбординг: setOnboardingStep(userId, step) — после каждого шага; setOnboardingCompleted — при завершении/пропуске.
- Редактирование контента: updateMenuItemContent, updateMenuItemTarget, переключение isActive, moveMenuItem — по действиям админа.

### Временное состояние (не персистентное)
- Состояние сцен (create-section, create-menu-item, edit-page-content и т.д.): title, draft, step — в ctx.wizard.state / scene; при выходе из сцены или перезапуске бота теряется только текущий незавершённый ввод.
- Навигационная сессия (navPrev, navCurrent): в ctx.session; используется только для «Назад» в рамках сессии.

### Отмена
- «Отмена» в сцене: scene.leave(), ответ «Действие отменено», клавиатура «В главное меню» / «Вернуться в админку». В БД ничего не откатывается и не удаляется — отменяется только текущий незавершённый шаг.

### Полное обнуление
- Отдельное действие: admin → «⚠️ Полностью обнулить бота» → подтверждение → wipeBotStructure + resetOnboarding. Удаляются все пункты меню и сбрасывается онбординг; приветствие возвращается к дефолтному. Отмена и полное обнуление разделены.

### Итог по персистентности
- Успешные действия сохраняются сразу; отмена не затирает сохранённые данные; полное обнуление — явное опасное действие с подтверждением.

## 4. New simplified product logic

- **Страница (раздел):** экран с контентом (текст; фото/видео/документ с подписью или только файл). Имеет parentId (null = на главной), может иметь дочерние страницы и кнопки.
- **Кнопка:** элемент меню, ведущий на страницу. Либо создаётся автоматически при «Добавить новый раздел» (новая страница + кнопка на текущей странице), либо создаётся отдельно при «Добавить новую кнопку» (кнопка → существующая страница, SECTION_LINK).
- **Главная страница:** root; контент из getWelcome, кнопки — getMenuItemsForParent(null).
- **Подраздел:** дочерняя страница (parentId = id текущей страницы); создаётся через «Добавить новый раздел» из редактора этой страницы.

Правила, отражённые в копи и сценариях:
- «Добавить новый раздел» = новая страница + кнопка к ней на текущей странице.
- «Добавить новую кнопку» = только кнопка, привязка к существующей странице.
- Подраздел = дочерняя страница внутри открытой; контекст задаётся section_hint_inside_page.

## 5. New ideal beginner flow (quick build)

1. Выбор основного языка (один раз).
2. Настройка главной страницы (одно сообщение: текст/фото/видео/документ).
3. Создание первого раздела: название кнопки → контент одним сообщением → сохранение.
4. Выбор: «Добавить ещё один раздел на главной» или «Предпросмотр и дальше».
5. Предпросмотр структуры, при необходимости — предупреждения.
6. Публикация → «Готово».

Дополнительные кнопки и тонкая настройка не входят в обязательный онбординг; они доступны в ручном режиме после публикации.

## 6. New advanced manual editing flow

- Вход: админ открывает бота, на любой странице видит «🛠 Настроить страницу» → редактор этой страницы.
- В редакторе: «Изменить содержимое страницы», «Добавить новый раздел», «Добавить новую кнопку», «Управление кнопками страницы», «Назад», «В главное меню».
- Управление кнопками: отдельный режим; по каждой кнопке — название, целевая страница, статус, порядок; действия: переименовать, изменить раздел назначения, вкл/выкл, выше/ниже, удалить.
- Создание раздела (вручную): как в онбординге — сначала название кнопки, затем контент одним сообщением; контекст «Новый раздел будет создан внутри: [страница]» задаётся через section_hint_inside_page / section_hint_on_root.

## 12. What now definitely works end-to-end

- `/start` и «🗂 В главное меню» открывают одну и ту же сохранённую главную (getWelcome + кнопки первого уровня).
- «↩️ Назад» с раздела/подраздела ведёт на страницу родителя с контентом (sendMenuPage по parentId).
- Кнопки разделов ведут на соответствующие страницы с контентом; SECTION_LINK ведёт на targetMenuItemId.
- У админа на каждой странице есть «🛠 Настроить страницу» для текущей страницы; у пользователя этих кнопок нет.
- После удаления страницы в редакторе открывается полная страница родителя, а не пустой список.
- Онбординг: язык → главная → раздел(ы) → предпросмотр → публикация; копи без «дополнительной кнопки» в быстром потоке.
- Полное обнуление: отдельная кнопка с ⚠️ и подтверждение; отмена не трогает сохранённые данные.

## 13. What to press in Telegram to verify

1. **Главная:** `/start` — должна открыться главная с приветствием и кнопками разделов; нажать «🗂 В главное меню» с любой страницы — тот же экран.
2. **Назад:** зайти в раздел → подраздел → «↩️ Назад» — снова раздел с контентом; ещё «↩️ Назад» — главная.
3. **Редактор:** зайти в раздел под админом → «🛠 Настроить страницу» — редактор этого раздела (содержимое, добавить раздел/кнопку, управление кнопками).
4. **Удаление страницы:** в редакторе раздела удалить дочернюю страницу и подтвердить — должен открыться экран родительского раздела с контентом и кнопками.
5. **Онбординг:** админ, пустое меню → «Запустить мастер» → пройти язык, главную, первый раздел → «Предпросмотр и дальше» → «Опубликовать»; проверить, что в текстах нет «дополнительная кнопка», есть «добавить ещё раздел или предпросмотр».
6. **Обнуление:** Админ-панель → «⚠️ Полностью обнулить бота» → подтверждение — бот пустой, онбординг сброшен.

## 14. How to run the automated checks locally

```bash
npm run test
```

Покрытие:
- Граф навигации и валидация (navigation-audit, navigation-integrity).
- Контракт клавиатур и callback_data (keyboards).
- Запуск полного аудита по БД (при наличии данных): в коде вызвать `await services.menu.runNavigationAudit({ requireRootContent: true })` и обработать массив ошибок (BROKEN_BUTTON_TARGET, SECTION_LINK_MISSING_TARGET, ORPHAN_PAGE и т.д.).

Подробности по тестам и интерпретации падений — в `tests/README.md`.
