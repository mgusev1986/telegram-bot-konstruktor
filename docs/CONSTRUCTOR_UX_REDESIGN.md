# UX Audit and Redesign: Конструктор бота

## 1. UX Audit — текущие проблемы

### 1.1 Путаница в понятиях
- **Раздел vs страница vs кнопка:** в интерфейсе используются «раздел», «страница», «пункт», «кнопка», «этот раздел» без единой модели. Админ не понимает, что раздел = страница с контентом, а кнопка = элемент меню, ведущий на страницу.
- **«Настроить страницу»** на корне — неочевидно, что это редактор той же страницы (главной). Нет явного «редактирование страницы: Главная».

### 1.2 Перегруженные экраны
- **Редактор страницы:** заголовок — только название страницы («Главная» или название раздела). Нет контекста «Редактирование страницы» и «Текущая страница: …», из-за чего легко потеряться при нескольких открытых экранах.
- **Управление кнопками:** длинный заголовок (hint_manage_buttons), список кнопок с «→ этот раздел» / «→ target» — для новичка не сразу ясно, что «этот раздел» значит «ведёт на саму эту страницу».

### 1.3 Несогласованность формулировок
- В подсказках: «Новый раздел — это новая страница с содержимым. Кнопка к ней создастся автоматически» — хорошо, но перед действием нет короткого «что произойдёт дальше».
- «Новая кнопка ведёт на уже существующий раздел» — корректно, но не сказано явно: «сначала название, потом выбор страницы».
- В сцене создания раздела первый шаг — выбор языка, без одной фразы «что такое раздел и что будет дальше».

### 1.4 Утечка технической лексики
- В коде/подсказках: «тип пункта», «SUBMENU», «SECTION_LINK», «родительская страница» — в UI пользователя лучше не показывать.
- Команда `/set_welcome` есть; при её использовании показывается подсказка про конструктор — ок, но редактирование корня должно быть только через «Настроить страницу» → «Изменить содержимое страницы».

### 1.5 Навигация и подтверждения
- Навигация в целом приведена к стандарту (Назад, В главное меню, Отмена, Вернуться к странице/в админку).
- После создания раздела/кнопки показывается «Вернуться к странице» / «Вернуться в админку» — хорошо; не везде есть короткое подтверждение «что принято» (в онбординге шага 1 это уже добавлено).

### 1.6 Восстановление после ошибок
- При отмене сцены или устаревшей кнопке есть сообщения и кнопки выхода. Не везде явно сказано «откройте нужную страницу заново» — в action_stale уже есть.

## 2. Новая информационная архитектура

### 2.1 Модель для пользователя
- **Страница (раздел)** = экран с контентом (текст, фото, видео, документ). У каждой страницы есть «содержимое» и список «кнопок», ведущих с этой страницы куда-то.
- **Кнопка** = элемент меню на странице. Ведёт либо на новую страницу (раздел), либо на уже существующую страницу.
- **Главная страница** = корневая страница; редактируется так же, как любая другая (через «Настроить страницу» → редактор страницы → «Изменить содержимое страницы»).

### 2.2 Иерархия экранов
1. **Корень (главная)** — контент + системные кнопки (Кабинет, Наставник, Сменить язык) + у админа: Админ-панель, Настроить страницу.
2. **Редактор страницы** — заголовок «Редактирование страницы», контекст «Текущая страница: [название]». Действия по порядку:
   - Изменить содержимое страницы
   - Добавить новый раздел
   - Добавить новую кнопку
   - Управление кнопками страницы
   - (если не корень) Удалить страницу
   - Назад, В главное меню
3. **Управление кнопками страницы** — отдельный экран: список кнопок (название → куда ведёт, статус), действия по каждой (переименовать, изменить раздел, вкл/выкл, выше/ниже, удалить), Назад, В главное меню.
4. **Визарды (сцены):** создание раздела, создание кнопки, редактирование контента — с коротким объяснением в начале и однозначными шагами.

### 2.3 Логика кнопок и экранов
- **«Изменить содержимое страницы»** — меняет контент текущей страницы (текст/медиа). Что дальше: отправить одно сообщение с новым контентом.
- **«Добавить новый раздел»** — создаётся новая страница + автоматически создаётся кнопка на текущей странице. Что дальше: название кнопки → отправить контент страницы одним сообщением.
- **«Добавить новую кнопку»** — создаётся только кнопка на текущей странице, привязка к существующей странице. Что дальше: название кнопки → выбор страницы из списка.
- **«Управление кнопками страницы»** — экран переименования, смены цели, порядка, вкл/выкл, удаления кнопок текущей страницы.

## 3. Новая иерархия кнопок и экранов (итог)

| Экран | Заголовок / контекст | Основные кнопки |
|-------|----------------------|------------------|
| Корень | Приветствие + меню | Контент, Кабинет, Наставник, Сменить язык, [Админ-панель], [Настроить страницу] |
| Редактор страницы | «Редактирование страницы» + «Текущая страница: [X]» | Изменить содержимое, Добавить раздел, Добавить кнопку, Управление кнопками, [Удалить страницу], Назад, В главное меню |
| Управление кнопками | «Управление кнопками страницы» + подсказка | Список кнопок + действия, Назад, В главное меню |
| Создание раздела | Объяснение + шаги | Язык → Название → Контент (одним сообщением), Назад, Отмена |
| Создание кнопки | Объяснение + шаги | Название → Выбор страницы, Назад, Отмена |
| Редактирование контента | Объяснение + «отправьте контент» | Отмена, (Вернуться к странице / в админку после сохранения) |

## 5. Что видит админ по шагам (после доработок)

1. **Открывает бота** → главная (контент + кнопки меню). У админа есть «Настроить страницу».
2. **Нажимает «Настроить страницу»** → экран «Редактирование страницы. Текущая страница: Главная» и кнопки: Изменить содержимое, Добавить раздел, Добавить кнопку, Управление кнопками, Назад, В главное меню.
3. **«Изменить содержимое страницы»** → короткое объяснение («Меняете контент этой страницы. Отправьте одним сообщением текст, фото, видео или документ») → сцена приёма контента → сохранение → «Главная страница обновлена» + Вернуться к странице / В админку.
4. **«Добавить новый раздел»** → короткое объяснение («Будет создана новая страница и кнопка на этой. Сначала введите название кнопки») → сцена: язык → название → контент одним сообщением → «Раздел создан. Кнопка привязана автоматически» + Вернуться к странице / В админку.
5. **«Добавить новую кнопку»** → короткое объяснение («Будет создана кнопка на этой странице. Введите название, затем выберите страницу») → сцена: название → выбор страницы → «Кнопка создана и привязана» + Вернуться к странице / В админку.
6. **«Управление кнопками страницы»** → экран со списком кнопок и действиями (переименовать, изменить раздел, вкл/выкл, выше/ниже, удалить), Назад, В главное меню.
7. **Назад** из редактора страницы → главная (если редактировали корень) или редактор родительской страницы.

## 6. Как тестировать переработанный конструктор в Telegram

1. **Сброс:** отправить `/start` — должна открыться главная.
2. **Вход в редактор:** на главной нажать «Настроить страницу». Должны быть заголовок «Редактирование страницы», строка «Текущая страница: Главная» и кнопки в порядке: Изменить содержимое, Добавить раздел, Добавить кнопку, Управление кнопками, Назад, В главное меню.
3. **Редактирование контента:** в редакторе нажать «Изменить содержимое страницы» → прочитать объяснение → отправить текст или фото с подписью → убедиться в сообщении «Главная страница обновлена» и кнопках «Вернуться к странице» / «Вернуться в админку».
4. **Создание раздела:** в редакторе нажать «Добавить новый раздел» → объяснение → выбор языка → ввод названия кнопки → отправка контента одним сообщением → «Раздел создан. Кнопка привязана автоматически» → вернуться в редактор или в админку.
5. **Создание кнопки:** в редакторе нажать «Добавить новую кнопку» → объяснение → ввод названия → выбор страницы из списка → «Кнопка создана и привязана» → вернуться.
6. **Управление кнопками:** «Управление кнопками страницы» → список кнопок с указанием «→ [страница]» / «→ этот раздел» → проверить переименование, смену раздела, вкл/выкл, порядок, удаление.
7. **Навигация:** из любого экрана проверить «Назад», «В главное меню», «Отмена» в сценах — без зависаний и потери контекста.
